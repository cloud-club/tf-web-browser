# 정리 : 브라우저는 어떻게 작동하는가

- 사용자 인터페이스 → 브라우저 엔진(데이터 보존) → 렌더링 엔진 → 네트워킹/자바스크립트해석기/UI백엔드

## 렌더링

- 파싱과 트리 생성
    - 브라우저는 HTML 문서를 받으면 파싱 처리를 진행합니다.
    - 파서는 받은 문서를 읽어서 문서 객체 모델 트리(DOM)를 생성합니다. DOM은 문서의 구조를 나타냅니다. 브라우저는 이 DOM 트리를 사용해 페이지에 콘텐츠를 렌더링합니다.
    - 브라우저는 HTML을 파싱해 DOM을 생성하면서 CSS를 처리해 CSS 객체 모델 (CSSOM)을 만듭니다. CSSOM은 DOM 요소에 적용될 CSS 스타일의 표현입니다.
- 렌더링의 주요 단계 : 레이아웃과 페인트
    - 레이아웃 단계에서 브라우저는 CSS에 정의된 스타일과 레이아웃 규칙을 기반으로 페이지에서 각 요소의 크기와 위치를 계산합니다.
    - 레이아웃을 완료하면 브라우저는 페인트 단계에 진입합니다. 페인트 단계에서 브라우저는 각 요소를 화면에 그립니다.
    - 이 처리 과정을 이해해야 애플리케이션 성능 최적화에 도움이 됩니다.
        - 예를 들어 페이지 안에 수많은 요소들이 존재하는 경우 브라우저는 레이아웃을 계산하는 데 많은 시간을 소요할 수 있습니다.
        - 페이지의 요소 수를 최소화하거나, CSS grid를 사용하거나, Flexbox를 사용해서 레이아웃을 단순화함으로써 애플리케이션 성능을 개선할 수 있습니다.

## 로딩

- 브라우저는 HTML 문서를 받으면 문서에 참조된 리소스를 로딩하기 시작합니다. 이 프로세스에서는 특히 로딩할 리소스가 많으면 병목이 될 수 있습니다.
- 로딩 프로세스를 이해해서 성능 최적화가 가능한데, 지연 로딩을 사용해 리소스가 필요할 때만 로딩할 수 있습니다. 또한 prefetching이나 preloading 같은 리소스 힌트를 사용할 수 있습니다.
